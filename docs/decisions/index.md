## Decision Lifecycle

1. **Need identified** — Agent or team identifies a decision requirement
2. **Decision request created** — Founder Decision Request issue opened
3. **Founder ratifies** — Founder approves and documents decision
4. **Decision logged** — Decision file added to decisions/YYYY/FD-YYYY-NNN.md
5. **Index updated** — This index is updated in the same PR
6. **Enforcement activated** — CI checks and governance rules enforced

---

## How to Use This Index

- **For agents:** Reference the decision ID (e.g., FD-2026-001) in all related work
- **For Founder:** Use this to track all decisions and their status
- **For auditing:** This is the authoritative record of all governance decisions
